# Gemini 2.5 Flash Lite 기본 분석 모델 설계

## 목표

AI 분석에서 `gemini-2.5-flash-lite`를 선택 가능한 모델로 추가하고, 분석 모델 설정이
새로 생성되거나 존재하지 않을 때 사용하는 기본 모델로 지정한다.

이미 `analysis_model_config`에 저장된 사용자의 모델 선택과 기존
`analysis_results.model_id` 값은 변경하지 않는다.

## 변경 범위

### 설정 화면

- 분석 모델 선택 목록에 `gemini-2.5-flash-lite`를 추가한다.
- 목록의 첫 항목으로 배치해 새 분석 설정을 추가할 때 기본 선택값으로 사용한다.
- 기존 설정 API에서 받은 `modelId`는 그대로 표시하므로 저장된 선택은 유지된다.

### 서버 기본값

- `analysis_model_config` 행이 없을 때 반환하는 기본 모델을
  `gemini-2.5-flash-lite`로 변경한다.
- 새 분석 설정을 생성하면서 `modelId`가 전달되지 않은 경우에도 같은 기본값을
  사용한다.
- 모델 ID가 `gemini`으로 시작하는 기존 API 키 해석 경로를 그대로 사용한다.

### 초기 데이터

- 개발 및 신규 환경용 시드 데이터의 분석 모델 기본값을
  `gemini-2.5-flash-lite`로 변경한다.
- 초기 스키마 구축용 분석 모델 설정도 새 기본값과 일치시킨다.
- 기존 운영 DB 행을 갱신하는 `UPDATE` 마이그레이션은 추가하지 않는다.

## 데이터 보존

- 기존 `analysis_model_config` 행은 수정하지 않는다.
- 기존 `analysis_results` 행의 `model_id`는 수정하지 않는다.
- 따라서 현재 `gemini-2.5-flash`, Claude 또는 GPT 모델을 선택한 분석 유형은
  사용자가 직접 변경하기 전까지 해당 모델을 계속 사용한다.

## 오류 처리

`gemini-2.5-flash-lite`는 기존 Gemini 모델과 동일한 공급자 및 API 키 해석 경로를
사용한다. 별도의 오류 처리 분기는 추가하지 않고 기존 분석 실행 오류 처리와 크론
감사 로그를 그대로 적용한다.

## 테스트

- 설정 화면에 `gemini-2.5-flash-lite` 옵션이 존재하고 새 설정의 기본 선택값인지
  검증한다.
- 설정 행이 없거나 새 행 생성 시 서버 기본값이 `gemini-2.5-flash-lite`인지
  검증한다.
- 시드 데이터가 새 기본값을 사용하는지 검증한다.
- 기존 모델 ID를 전달한 설정 업데이트가 해당 값을 유지하는 기존 동작을 검증한다.
- 타입 검사와 관련 테스트를 실행한다.

## 제외 범위

- 기존 운영 설정의 일괄 변경
- 기존 분석 결과의 모델 ID 변경
- 모델별 프롬프트 또는 생성 파라미터 조정
- 공급자/API 키 구조 변경
